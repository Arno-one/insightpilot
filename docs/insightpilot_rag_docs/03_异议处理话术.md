# InsightPilot 异议处理话术知识库

doc_id: objection_handling_v1
适用系统: InsightPilot AI 企业运营参谋系统
适用场景: 客户流失风险解释、销售跟进建议、任务话术生成
文档版本: v1.0

## OBJ-001 客户说价格太贵

客户信号：

- “你们这个价格有点高。”
- “别家比你们便宜。”
- “我们预算没这么多。”

风险判断：

- 如果客户已经认可价值，只是价格有压力，属于可挽回风险。
- 如果客户只关注价格，不愿讨论需求和效果，成交质量较低。
- 如果报价后超过 7 天客户仍围绕价格犹豫，应标记为中风险。

跟进目标：

- 不要马上降价。
- 先确认客户觉得贵的原因。
- 把价格问题转化为投入产出问题。

推荐话术：

我理解您会关注预算。我们可以先不急着讨论折扣，我想先确认一下，您觉得价格高主要是因为总预算有限，还是还没有完全看到这套系统能帮团队减少哪些销售管理损失？

如果客户说预算有限：

可以回应：

如果预算是当前主要限制，我们可以先从一个销售小组试点，不需要一开始全团队铺开。先验证系统能否发现高风险客户、减少漏跟进，再决定是否扩大使用。

如果客户说竞品便宜：

可以回应：

竞品价格低可能是一个优势，不过我们建议重点比较三件事：风险识别是否可解释、AI 任务是否需要人工确认、经营日报能不能直接辅助管理决策。如果只是便宜，但不能解决客户流失预警，后续管理成本可能还是会存在。

Agent 建议动作：

- 生成“价格异议处理”任务。
- 建议销售员补充客户预算范围。
- 对高金额客户，建议销售主管介入。

## OBJ-002 客户说没有预算

客户信号：

- “今年没有预算。”
- “我们下个季度再说。”
- “老板还没批预算。”

风险判断：

- 没有预算不一定等于无需求。
- 如果客户有明确痛点但预算周期未到，应设置再激活任务。
- 如果客户没有痛点也没有预算，应降低优先级。

跟进目标：

- 确认是否是真没预算，还是价值未被认可。
- 确认预算申请周期。
- 争取进入下一轮预算讨论。

推荐话术：

明白，那我想确认一下，当前是完全没有销售数字化相关预算，还是预算需要等到下个周期审批？如果是后者，我们可以先帮您梳理当前销售管理中的风险点，方便您后续向老板申请预算时有依据。

如果客户表示下季度再看：

可以回应：

可以的。那我建议我们先约一个轻量复盘，把目前客户跟进延迟、报价后无回应、销售主管介入不及时这些问题整理出来。这样到下季度讨论预算时，不会只是讨论买不买系统，而是讨论这些损失怎么解决。

Agent 建议动作：

- 创建预算周期回访任务。
- 截止时间设置在客户预算周期前 2-4 周。
- 如果客户金额高，应保留为中风险观察客户。

## OBJ-003 客户说需要再考虑

客户信号：

- “我们再考虑一下。”
- “我先内部讨论。”
- “晚点给你回复。”

风险判断：

- “再考虑”通常不是明确下一步。
- 如果没有约定具体反馈时间，容易进入沉默流失。
- 报价阶段出现“再考虑”，应要求销售员补充下一步计划。

跟进目标：

- 明确客户还在考虑什么。
- 明确下次沟通时间。
- 明确决策人和决策标准。

推荐话术：

可以的。为了不打扰您，我想确认一下，您这边主要是想再比较产品能力、内部预算，还是需要和其他同事确认？我们可以约一个具体时间，我针对您最关心的点准备更聚焦的信息。

如果客户不愿给时间：

可以回应：

没问题。那我先把今天沟通里您关注的几个点整理成一页说明发您，您看完后如果方便，我们再用 15 分钟确认是否适合继续推进。

Agent 建议动作：

- 创建“明确决策标准”任务。
- 如果 7 天内无回应，升级为中风险。
- 如果客户金额高，建议主管查看沟通记录。

## OBJ-004 客户说已经在看竞品

客户信号：

- “我们也在看其他系统。”
- “另一个供应商也给了方案。”
- “竞品价格更低。”

风险判断：

- 竞品介入是明确风险信号。
- 如果客户仍愿意沟通，说明还有比较机会。
- 如果客户只讨论价格，应回到价值和差异点。

跟进目标：

- 找出客户比较维度。
- 突出 InsightPilot 的可解释风险分析、人工确认和经营闭环。
- 避免攻击竞品。

推荐话术：

正常的，企业选系统确实应该比较。为了帮您更快判断，我建议我们不只比功能清单，而是比三个实际场景：能不能提前发现高风险客户、AI 建议能不能经过主管确认再落地、老板每天能不能看到可执行的经营报告。

如果客户强调竞品功能更多：

可以回应：

功能多不一定代表团队会用起来。InsightPilot V1 更强调销售风险闭环：发现风险、解释原因、生成任务、主管确认、销售跟进。这个链路如果跑通，管理价值会更直接。

Agent 建议动作：

- 创建“竞品比较跟进”任务。
- 推荐销售主管介入。
- 检索产品资料中的核心差异点。

## OBJ-005 客户长时间不回复

客户信号：

- 报价后 7 天无回应。
- 方案发送后 5 天无回应。
- 多次消息已读不回。
- 电话无人接听。

风险判断：

- 长时间不回复是客户流失的核心信号。
- 报价后无回应风险高于普通沟通后无回应。
- 高金额客户无回应应优先处理。

跟进目标：

- 用低压力方式重新建立连接。
- 不重复发送催促信息。
- 提供一个明确且容易回应的问题。

推荐话术：

您好，我这边不是催您做决定，只是想确认一下之前的方案是否还有继续评估的必要。如果现在优先级有变化也没关系，我可以按您的节奏调整后续沟通。

如果客户仍不回复：

可以发送：

我先把这个需求暂时标记为后续跟进，避免频繁打扰您。如果之后销售管理、客户流失预警或经营日报这块重新进入计划，您可以随时联系我。

Agent 建议动作：

- 第一次无回应：创建轻量跟进任务。
- 第二次无回应：建议更换沟通方式。
- 高金额客户第三次无回应：建议主管介入或暂时关闭。

## OBJ-006 客户担心 AI 不可靠

客户信号：

- “AI 判断会不会乱来？”
- “如果 AI 创建错任务怎么办？”
- “我们不能让 AI 直接操作业务。”

风险判断：

- 这是合理担忧，不应强行反驳。
- 如果客户关注安全和审批，说明客户有企业级使用意识。

跟进目标：

- 强调系统采用规则引擎打底。
- 强调 AI 只生成解释和建议。
- 强调关键动作必须人工确认。

推荐话术：

您的担心非常合理，所以 InsightPilot 不会让 AI 直接替团队做关键业务决策。客户风险分由规则引擎计算，AI 负责解释原因和生成建议。真正创建销售任务前，必须经过销售主管确认。

补充说明：

系统会记录 Agent 每一步看了什么数据、调用了什么工具、生成了什么建议。这样即使建议不合适，也能追溯原因并调整规则。

Agent 建议动作：

- 检索销售 SOP 中的人工确认规则。
- 向客户解释 Agent 执行日志和审批机制。
- 如果客户是企业版潜在客户，推荐演示审计日志。

## OBJ-007 客户担心数据安全

客户信号：

- “我们的客户数据能不能隔离？”
- “销售员会不会看到不该看的客户？”
- “AI 会不会读取全公司数据？”

风险判断：

- 数据安全是企业客户的核心顾虑。
- 如果客户主动询问权限，说明其组织管理较成熟。

跟进目标：

- 说明 RBAC 权限控制。
- 说明销售员只能看到自己的客户。
- 说明 Agent 工具调用也受权限控制。

推荐话术：

系统会通过 RBAC 控制不同角色的数据访问范围。老板可以看全局经营数据，销售主管看团队数据，销售员只看自己负责的客户。Agent 调用工具前也会检查当前用户权限，不会因为是 AI 就绕过权限体系。

Agent 建议动作：

- 推荐演示老板、主管、销售员三个角色视角。
- 对企业客户重点说明 Agent 工具权限控制。
- 如果客户仍担心，可建议先用模拟数据试点。
