# InsightPilot RAG 原始知识库文档

这些文档用于 `InsightPilot：AI 企业运营参谋系统` 的 V1 知识库建设。

V1 的 RAG 目标不是做泛问答，而是增强 Agent 在客户流失风险分析中的建议质量。建议先把这些 Markdown 文档入库到 Milvus，再基于同类内容补充 QA 问答对，提高精确命中率。

## 文档清单

- `01_sales_sop.md`：销售 SOP，覆盖客户阶段、跟进节奏、风险动作和任务标准。
- `02_product_pricing.md`：产品资料与价格策略，覆盖套餐、卖点、适配客户和让利边界。
- `03_objection_handling.md`：异议处理话术，覆盖价格贵、没预算、竞品比较、暂缓决策等常见场景。
- `qa_generation_guide.md`：QA 问答对制作建议，帮助你把原文档加工成更利于检索的问答数据。

## 建议入库方式

1. 先入库原始 Markdown 文档。
2. 再把同类 QA 问答对作为独立文档或 `source_type=qa` 数据入库。
3. 问答对里保留 `doc_id`、`section_id`、`tags`，方便回答时追溯来源。
4. 对高频问题做多问法改写，比如“客户嫌贵怎么办”“客户说预算不足怎么回”“报价后不回复怎么办”都指向同一类知识。

## 推荐检索策略

- 用户问自然语言问题时，先做 query rewrite。
- 优先检索 QA 问答对。
- QA 未命中或分数不足时，再检索原始文档片段。
- Agent 生成建议时必须引用 SOP 或话术来源，避免凭空发挥。
