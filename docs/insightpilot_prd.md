# InsightPilot 产品需求文档 PRD

版本：v1.0
项目名称：InsightPilot
项目定位：面向中小企业的 AI 企业运营参谋系统
核心主线：基于 CRM 数据自动识别客户流失风险，并通过 Agent、RAG、人工审批和销售任务形成业务闭环。

## 一、问题陈述

中小企业销售团队常见问题不是没有客户数据，而是数据没有被及时转化为经营动作。客户报价后长时间无人跟进、销售员遗漏高价值客户、销售主管无法及时发现风险、老板只能月底看滞后的报表，这些都会导致客户流失。

传统 CRM 更偏记录系统，能存客户、商机和跟进记录，但很少主动告诉团队“哪些客户快流失了、为什么、下一步应该谁去做什么”。普通 AI 聊天助手又缺少权限、安全、审批、数据追踪和业务闭环，难以作为企业系统落地。

InsightPilot 要解决的问题是：

- 自动发现客户流失风险。
- 解释风险原因，并给出可执行建议。
- 将 AI 建议转成需要人工确认的任务草稿。
- 让销售主管确认后再分配给销售员。
- 让老板每天看到清晰的经营日报。
- 让 Agent 每一步执行都有日志、权限和来源追踪。

## 二、产品目标

V1 目标：

- 跑通从 CRM 数据到客户风险识别、AI 建议、人工审批、销售任务、经营日报的完整闭环。
- 展示企业级 Agent 系统的可靠性：RBAC、JWT、人工确认、Agent 执行日志、RAG 来源引用。
- 用内置模拟数据保证项目开箱可演示。
- 技术上采用模块化单体，为后续多服务、多租户和 SaaS 化预留空间。

完整终局目标：

- 从“销售风险分析工具”逐步演进为“企业运营参谋平台”。
- 支持销售、客户成功、运营、财务、人效等多业务域分析。
- 支持多租户 SaaS、企业知识库、多 Agent 协作、数据导入、自动化运营策略、评估闭环和可观测平台。
- 成为一个简历和作品集中可以完整讲清业务价值、架构设计和工程深度的项目。

## 三、V1 范围

V1 聚焦一条主线：

```text
CRM 模拟数据
  → 风险规则引擎打分
  → LangGraph Agent 生成解释和建议
  → RAG 检索销售 SOP / 产品资料 / 异议话术
  → 生成待审批任务草稿
  → 销售主管人工确认
  → 创建正式销售任务
  → 生成经营日报
```

V1 必做模块：

- 登录与权限：JWT + RBAC。
- SaaS 工作台前端：真实登录页、权限菜单、经营驾驶舱。
- CRM 基础数据：客户、联系人、商机、跟进记录。
- 风险分析：代码内置规则、风险快照、风险证据。
- Agent 编排：LangGraph 风险分析图、日报生成图。
- RAG 知识库：Milvus 双 Collection、QA + 文档切片。
- 人工审批：AI 任务草稿必须主管确认。
- 销售任务：审批通过后创建正式任务。
- 经营报告：日报为主，周报预留。
- 异步任务：Redis + RQ，支持 RAG 入库、风险扫描、报告生成。
- 可观测：Agent Run、Agent Step、RAG Trace、检索命中记录。

V1 不做：

- 完整多租户注册和租户后台。
- 计费套餐系统。
- 复杂工作流编辑器。
- 第三方 CRM 深度集成。
- 移动端 App。
- 语音外呼、自动发邮件、自动联系客户。
- 财务、人事、库存等非销售域分析。

## 四、角色与权限

V1 只做 3 个角色。

### 老板 owner

目标：快速掌握全局经营风险。

能力：

- 查看全局经营驾驶舱。
- 查看所有客户风险。
- 查看所有销售任务。
- 查看经营日报和周报。
- 查看 Agent 执行日志。
- 触发全局风险扫描。

### 销售主管 manager

目标：管理团队风险，确认 AI 任务草稿。

能力：

- 查看团队客户和团队风险。
- 触发团队风险扫描。
- 审批 AI 生成的任务草稿。
- 分配任务给销售员。
- 查看团队销售任务。
- 查看团队经营报告。

### 销售员 salesperson

目标：处理自己负责客户和任务。

能力：

- 查看自己负责的客户。
- 查看自己的任务。
- 填写跟进记录。
- 查看 AI 给自己的跟进建议。
- 更新任务状态。

权限原则：

- 前端控制菜单和按钮可见性。
- 后端控制接口访问权限。
- Agent Tool 调用前也必须检查权限。
- AI 不能因为是 Agent 就绕过 RBAC。

## 六、核心功能设计

### 1. 登录与权限

功能：

- 用户输入账号密码登录。
- 后端返回 JWT、用户信息、角色列表、权限列表。
- 前端根据权限渲染菜单和按钮。
- 后端使用依赖注入校验接口权限。
- Agent Tool 执行前校验 required_permissions。

验收标准：

- 未登录访问业务页面会跳转登录。
- token 过期后自动退出。
- 销售员不能访问全局客户风险接口。
- 销售员不能审批自己负责客户的 AI 任务草稿。

### 2. 经营驾驶舱

功能：

- 今日新增客户数。
- 今日有效跟进数。
- 今日高风险客户数。
- 销售漏斗阶段分布。
- 待主管确认 AI 任务数。
- 高风险客户 Top 5。
- 今日经营摘要。

验收标准：

- 老板看到全局数据。
- 销售主管看到团队数据。
- 销售员不显示全局经营驾驶舱，只显示个人任务摘要。

### 3. 客户风险中心

功能：

- 客户风险列表。
- 风险等级筛选。
- 客户阶段筛选。
- 负责人筛选。
- 风险原因展示。
- 规则命中明细展示。
- AI 建议和推荐话术展示。

验收标准：

- 风险分由规则引擎计算。
- 风险解释由 LLM 基于规则结果生成。
- 建议内容引用 RAG 来源。

### 4. AI 任务审批台

功能：

- 查看 AI 任务草稿。
- 查看风险证据。
- 查看推荐话术。
- 批准、驳回、修改后批准。
- 审批通过后创建正式销售任务。

验收标准：

- AI 不能直接创建正式任务。
- 审批记录完整保存。
- 审批通过后任务关联 approval_id。

### 5. 销售任务

功能：

- 查看我的任务。
- 查看团队任务。
- 更新任务状态。
- 填写执行结果。
- 展示任务来源和推荐话术。

验收标准：

- 销售员只能看自己的任务。
- 主管可以看团队任务。
- 任务必须能追溯到审批记录或人工创建人。

### 6. RAG 知识库

功能：

- 原始文档入库。
- QA 问答对入库。
- 双 Collection 检索。
- Query Rewrite。
- 稠密向量 + BM25 混合检索。
- RRF 融合。
- CrossEncoder 精排。
- Token 预算控制。
- 去重。
- 来源标记。
- Trace 日志。

验收标准：

- QA 数据可以逐行 JSONL 入库。
- 原始文档按 chunk_size=500、overlap=50 切片。
- QA 使用 search_text 做 BM25。
- 检索结果包含 doc_id、section_id、source_type。

### 7. LangGraph Agent

V1 至少实现两个图：

- 风险分析图。
- 经营日报图。

风险分析图：

```text
load_crm_data
  → calculate_rule_risk
  → retrieve_sales_knowledge
  → generate_risk_reason
  → generate_task_draft
  → create_approval_record
```

经营日报图：

```text
load_metrics
  → load_top_risks
  → retrieve_report_sop
  → generate_report
  → save_business_report
```

验收标准：

- 每个节点写入 agent_step。
- 整体执行写入 agent_run。
- 失败时保存 error_message。
- 需要人工确认的动作进入 awaiting_approval。

### 8. 异步任务

功能：

- RAG 入库任务。
- 批量风险扫描任务。
- 经营日报生成任务。

技术：

- Redis。
- RQ。
- APScheduler 轻量定时触发。

验收标准：

- 用户点击触发风险扫描后 HTTP 请求不被长时间阻塞。
- Worker 执行状态可查询。
- 定时任务可以每天自动触发风险扫描和日报生成。

## 九、测试决策

测试原则：

- 测外部行为，不测实现细节。
- 风险规则必须可重复、可解释。
- Agent 节点允许 mock LLM，但必须验证状态流转和落库结果。
- RAG 检索需要有固定评估集。

V1 优先测试模块：

- RBAC 权限校验。
- 风险规则引擎。
- 审批到任务创建流程。
- RAG JSONL 入库校验。
- LangGraph 风险分析流程。
- 经营日报生成流程。

关键测试场景：

- 销售员不能查看全局风险。
- AI 任务草稿未审批不能创建正式任务。
- 报价后 14 天无回应客户被识别为高风险。
- 竞品介入客户命中对应规则。
- RAG 检索返回来源信息。
