# InsightPilot 产品需求文档 PRD

版本：2026-07-01
产品定位：面向中小企业销售运营场景的 AI 经营参谋系统

## 1. 产品要解决什么问题

很多销售团队并不缺 CRM 数据，真正缺的是“把数据及时变成经营动作”的能力。

典型问题包括：

- 客户报价后长时间没人跟进
- 高价值客户风险上升，但主管发现太晚
- 销售员每天很忙，却不清楚先处理谁
- 老板只能看滞后的结果，不能尽早干预
- 普通聊天式 AI 给得出建议，但不能接入权限、审批和业务闭环

InsightPilot 的目标不是做一个聊天机器人，而是做一个能把“风险识别 -> 建议生成 -> 审批确认 -> 任务落地 -> 报告复盘”串起来的企业 Agent 系统。

## 2. 当前阶段目标

当前阶段的产品目标已经从“做出骨架”升级为“把 V1 主链路做成一个可演示、可解释、可审计的 Demo”。

当前 V1 已经完成的产品闭环：

```text
CRM 数据
  -> 风险规则识别
  -> RAG 检索相关销售知识
  -> LLM 生成解释、建议和话术
  -> 主管审批 AI 草稿
  -> 创建正式销售任务
  -> 汇总到经营日报
  -> 通过 Agent Trace 回看执行链路
```

## 3. 产品原则

### 3.1 风险判断必须可解释

风险分由规则引擎决定，不让 LLM 直接拍脑袋打分。

### 3.2 AI 不能直接执行关键动作

AI 可以生成草稿，但正式任务必须经过主管审批。

### 3.3 每一步都要能追踪

不仅要有结果，还要能回看：

- Agent 运行步骤
- RAG 检索来源
- 审批是否通过
- 最终是否落成任务

### 3.4 优先做业务闭环，不追求表面功能数量

当前阶段最重要的不是堆很多页面，而是让核心主链路可信、顺畅、可演示。

## 4. 用户角色

### 4.1 老板 `owner`

关注点：

- 今天有没有高风险客户
- 哪些客户最值得介入
- 审批和执行是否卡住
- 日报是否能快速读懂

### 4.2 销售主管 `manager`

关注点：

- 团队里哪些客户风险最高
- AI 建议是否合理
- 哪些任务应该通过、驳回或改后通过
- 销售执行有没有推进

### 4.3 销售员 `salesperson`

关注点：

- 自己今天该跟进谁
- 系统建议自己怎么说、怎么做
- 主管批下来的任务有哪些

## 5. 当前版本已实现的产品能力

### 5.1 登录与权限

- 账号密码登录
- JWT 会话
- 基于角色的菜单与接口权限控制

### 5.2 经营驾驶舱

- 汇总今日高风险客户
- 汇总待审批动作
- 汇总任务与日报信号
- 面向老板和主管做快速经营判断

### 5.3 风险中心

- 查看风险客户列表
- 查看风险等级、风险原因、AI 建议
- 查看风险是否已进入审批或任务阶段

### 5.4 AI 审批台

- 查看 AI 生成的任务草稿
- 审批通过
- 审批驳回
- 修改后审批通过

### 5.5 销售任务

- 查看正式销售任务列表
- 查看优先级、负责人、截止时间和状态

### 5.6 经营报告

- 生成经营日报
- 查看摘要与行动建议

### 5.7 Agent Trace

- 查看 Agent Run
- 查看步骤时间线
- 查看 RAG 命中来源

### 5.8 RAG 评估

- 运行评估
- 查看检索指标

### 5.9 风险扫描 `LangGraph`

- 已将风险扫描迁移为真实图编排
- 已保留原有审批、任务、Trace 主链路行为
- 已让 Worker 退回到轻量入口职责

### 5.10 经营日报 `LangGraph`

- 已将经营日报迁移为真实图编排
- 已保留原有报表落库和 Trace 主链路行为
- 已让 Worker 退回到轻量入口职责

## 6. 当前版本不做什么

为了让产品边界更清楚，当前版本明确不做这些内容：

- 多租户注册与租户后台
- 计费系统
- 复杂工作流编排器
- 移动端 App
- 第三方 CRM 深度同步
- 自动联系客户这类高风险动作

## 7. 当前最重要的用户故事

1. 作为老板，我希望一登录就能看到今天最值得关注的高风险客户和审批积压情况。
2. 作为销售主管，我希望系统能自动识别风险客户，并给出可审核、可落地的任务草稿。
3. 作为销售主管，我希望 AI 的建议必须先经过我确认，不能直接打到销售一线。
4. 作为销售员，我希望收到的任务不只是标题，还能带上风险原因和推荐话术。
5. 作为老板或主管，我希望能回看 Agent 每一步做了什么，确保系统不是黑盒。

## 8. 当前版本验收标准

只要下面这些都成立，就说明当前 V1 具备演示价值：

1. 可以使用内置账号正常登录。
2. 可以触发风险扫描。
3. 可以看到中高风险客户、原因和建议。
4. 可以在审批台处理 AI 草稿。
5. 审批通过后可以看到正式销售任务。
6. 可以生成并查看经营日报。
7. 可以在 Agent Trace 中看到执行步骤和 RAG 来源。
