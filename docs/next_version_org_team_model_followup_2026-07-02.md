# 下个版本补充需求：真实组织架构与团队模型

## 背景

- 当前经营报告页已经支持“按归属人继续下钻”
- 当前页面里的“团队视角”先按报告中的负责人聚合结果承接
- 这能满足本版本的 V1 经营分析需求，但还不等于真实组织架构团队

## 下个版本必须补的内容

- 新增真实组织架构模型
  - 例如团队表、团队成员关系、团队负责人关系
- 为用户补充稳定的团队归属字段
  - 避免继续用“同租户可见”近似代替“团队可见”
- 把 CRM、风险、任务、报告中的 `team` 口径统一到真实团队模型
- 把经营报告中的“团队下钻”从聚合视图升级为真实团队维度
- 为权限体系补齐“本人 / 团队 / 全部”的真实数据边界

## 当前版本为什么先不做

- 现有系统还没有独立的团队/组织架构数据库模型
- 如果这轮直接硬做“真实团队下钻”，会把经营报告切片拖成组织架构改造项目
- 当前更合适的做法，是先把“归属人下钻”做实，再把真实团队模型收进下个版本主需求

## 备注

- 本文档用于明确记录下个版本的迭代要求
- 当前版本的报告页团队视角，均应理解为“负责人聚合团队视角”，不是正式组织团队视角
